GIT-SVN

Remote commands comparison

Git Git-svn Svn
git clone REPO_URL git svn clone REPO_URL svn checkout REPO_URL
git pull origin master git svn rebase svn update
git push origin master git svn dcommit svn commit

Hey, you have only showed me three commands, where is all other awesome local git commands?

Good question!

As I have described before, git-svn creates a local git server.

Which means, you have access to all local git commands

  • git commit
  • git stash
  • git checkout -b localBranchName
  • git log
  • git cherry-pick commit-id
  • ...

Pros*

  • Better merge strategies (goodbye odd conflicts)
  • Stop the current task, to fix a production bug, with no code losses
  • Move a single commit all over around branches (cherry-pick üòç)
  • Offline access to commit logs, or blame tools (üòà)
  • Git commands for those of you who are familiar with that

* In my humble opinion...

Cons*

  • Some workflows act different than a pure GIT environment
  • Some other workflows must be avoided at all cost!
  • Long time to checkout the entire code base for the first time
  • Poorly documented in the internet
  • No GUI interface (for those of you who are affraid of the terminal)

* In my humble opinion...

The practical part

Initial Setup of SVN

For this guide, I'll be using a dockerised subversion server, so everyone could follow it in the same way.

For running the container, run the following commands in the root directory of this repo: 


$ cd ~/git/practical-guide-to-git-svn/svn-server
$ docker-compose up

Initial Setup of SVN

Wait around 10min until the following command:


$ svn info http://localhost/svn/guideRepo

Ouputs something like this:

Initial Setup of SVN

Now, run the project setup script: 


$ cd ~/git/practical-guide-to-git-svn/svn-server
$ sh svnProjectSetup.sh

And now, you should have a basic project ready to follow this guide.

Checkout the code

To checkout the code, go to a new terminal console, and run the following commands:


$ cd ~/
$ mkdir -p git
$ cd git
$ git svn clone http://localhost/svn/guideRepo \
    -T trunk/myProject \
    -b branches/myProject \
    -t tags/myProject \
    --no-minimize-url \
    myProject
